Introduction

At the start of this semester, I honestly didn’t think I would ever get along with CSS. I’d tried using it before, but I never really understood how all the different parts of a website connected or why certain things behaved the way they did. I expected CSS to be fun and creative, and sometimes it is — at least when you’re not fighting with rotating headings, animations, or effects that randomly break everything. Before this module, my confidence with HTML and CSS was pretty low, but I was hoping that this time I’d finally learn it properly instead of guessing my way through. Even with the doubts, I came into the module with a clear goal: by the end of the term, I wanted to build a proper portfolio website that I could actually publish and show to employers. Something that looked clean, felt like “me,” and showed the skills I’ve been developing.

Module Experience

I really liked this module — it’s honestly my favourite one so far in first year. The lessons were actually enjoyable, which made a huge difference, and I found the things we were learning genuinely interesting rather than just something I had to get through. I enjoyed the process of learning new techniques, especially when it came to aligning different tags properly and writing cleaner, more organised code. It felt good to see my work improve as I understood more. I was surprised that CSS can do so much more than I originally thought. Before this module, I saw it as something simple that just changed colours and fonts, but now I realise how powerful it actually is. You can create animations, 3D effects, responsive layouts, and so many small details that completely change how a website feels. At the same time, I also learned how easy it is to break everything with one wrong line, which taught me to be more patient and to test things properly. Looking back, I can see how much my confidence has grown. At the start, I didn’t think I’d ever understand CSS properly, but now I feel like I can build something from scratch and actually know what I’m doing. I’ve learned how to debug issues, how to structure my code better, and how to think about design in a more intentional way. Overall, this module has made me more confident as a developer and more excited about web design than I expected.

My Website Over the Term

The first thing I did was to setup the grid, after this I slowly added more components like the nav, heading, footer and main. Once the basic structure was ready, I styled the header and then added a bit of content to main. I created a CV button and added it to the navbar to make it look like a legit portfolio and shortly after that I imported the file so that the button actually downloads my CV. I downloaded some icons which I put inside the footer, after which I started to add some responsiveness to headings, text, anchor tags etc. So that the website still looked nice even when the screen was smaller. One big change was when I added mobile optimisation, this was a crucial feature as it allows anybody to be able to view my portfolio. I then proceeded to add some content to projects (just some projects from college and the python one that I did in the first semester) both desktop and mobile. I then added a form to the contact page so that people can email me, I used “mailto:” as a workaround from using JS for now, I will make it send emails without using “mailto:” in the future. I implemented a sliding navbar to the mobile version after the “Animations” lecture, this made the overall feel of the website more professional. I fixed a few problems like the navbar not filling out the whole width when landscape and removed unnecessary code like a hover effect on a card for the mobile version. I animated the project cards to straighten out and increase the opacity when scrolling down the page. I also decided that I didn't like the footer being in the middle of the page because when the page becomes a bit longer like in the projects page, the footer goes too far down the page. So, I decided to make it fixed to the left side of the page. I added an animated background which was originally squares that moved up the page and became rounder the more they travelled, but i didn’t like that so i made the “border-radius" larger and a bit more random so that it looks a bit more like a blob or a bubble. I also added a gradient background which is also animated, it changes colour every few seconds which makes the website look more appealing rather than boring. I added some shadow behind the headings inside the header, this also took a lot of time and caused a lot of problems which i didn't realise until I had to comment out all of the animations that were for the background in order for me to realise the weird overflowing was because i didn't set a width to one part inside of the header which stretched across the width of 2 monitors. Adter I fixed that, everything worked as expected and I haven’t had any problems. The last thing i did was adjusting some space, colours and sizes.

Inspiration

My initial idea for this project was honestly just to copy someone else’s portfolio, change the colours, swap the text, and call it a day. But after looking at a few examples online, I realised quickly that doing that wouldn’t teach me anything, and it wouldn’t feel like my work. The more I looked at other portfolios, the more I started picking out individual features I liked instead of trying to copy a whole design. I’d see a cool heading on one site, a nice layout on another, or a clean navigation bar somewhere else, and I tried to bring those ideas together in my own way. A few portfolios caught my eye:
- Portfolio 1
- Portfolio 2
- Portfolio 3
Most of the ideas I used came from portfolios I found on that website. Like I mentioned before, whenever I saw a feature I liked, I’d usually find a way to add my own version of it into my site sooner or later. I got my colour palette using Colour Palettea , this made it easy to choose colours that would go well with each other. I did end up tweaking the colours a bit in the end so that they fit in with the animated background and shadows that i added. For the fonts I used Google Fonts as its free and widely used across the world. It was easy to find a good font that would suit my portfolio.
